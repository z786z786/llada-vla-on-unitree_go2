# 首轮采集计划

这份说明与当前的键盘 Go2 采集闭环保持一致。

## 目标

先收集足够多、足够干净的短片段，回答三个问题：

- 相机流是否和操作员意图对齐
- `vx`、`vy`、`wz` 是否能表达我们关心的行为
- 以当前基线接口来看，这批数据是否可学

## 每个 Session 必填元数据

开始采集前，在启动命令里设置：

- `scene_id`
- `operator_id`

每条 episode 的 `instruction` 在片段结束后确定。

## 指令集合

只使用受控集合：

- `go forward`
- `move backward`
- `strafe left`
- `strafe right`
- `stand up`
- `lie down`
- `turn left`
- `turn right`
- `stay still`

## 采集流程

1. 启动 `go2_collector`
2. 用键盘遥操作把机器人移动到起始位姿
3. 按 `R` 启动一个采集段
4. 执行一个明确、单一的目标动作
5. 结束该段并保存有效数据
6. 若片段质量差，立刻丢弃并重采

显式采集段之外的帧不会写入磁盘。

## 最低覆盖目标

建议至少覆盖：

- 多个 session
- 多个 scene
- 所有核心动作指令
- 既有干净样本，也有少量轻微扰动样本

## 需要暂停采集并修复流水线的情况

若出现以下情况，应先停下来修系统：

- 图像文件缺失
- 指令标签错误
- 动作曲线长期贴零
- 明显图像延迟或时间戳陈旧
- 正常遥操作时仍频繁触发 safety fault
- 数据极端偏向单一 instruction
