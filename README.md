# llada-vla-on-unitree_go2

Local Go2 data collection and offline validation pipeline for a staged LLaDA-VLA workflow.

This repository keeps two collection regimes side by side:

- stage 1: canonical motion mapping such as `go forward -> [vx, vy, wz]`
- stage 2: visually necessary tasks where `instruction + image (+ optional state)` must determine the action

The current workspace provides:

- `native/build/go2_collector_stage1`: dedicated first-stage `single_action` collector
- `native/build/go2_collector`: general collector with both legacy motion capture and trajectory-based semantic tasks
- `scripts/sanity_check_dataset.py`: HTML replay and sanity report
- `tools/validate_bc_dataset.py`: offline trainability validator
- `tools/convert_llada_vla_dataset.py`: raw-session to manifest converter
- `tools/llada_vla_baseline.py`: lightweight baseline
- `tools/llada_vla_baseline_v2.py`: multimodal diagnostic baseline

## Repository layout

```text
go2_vla_collector/
  native/
  scripts/
  tools/
  docs/
  data/          # generated by the general collector
  data_stage1/   # generated by the stage1 collector
  outputs/       # generated reports and baseline outputs
```

Generated directories are intentionally ignored by git.

## Task interface

Current learning target:

- input: `instruction + front RGB + optional robot state`
- target: `control_action(vx, vy, wz)`

The collector stores both:

- `raw_action`: operator-side command trace
- `control_action`: training/deployment target

At the moment these are equal on `vx`, `vy`, `wz`, but the split is preserved so future remapping does not require changing the rest of the pipeline.

## Session schema

The general collector writes sessions under `data/`.
The dedicated stage1 collector writes sessions under `data_stage1/`.

Example layout:

```text
data/<session_id>/
  index.json
  episodes/
    ep_000001.json
  images/
    ep_000001/
      20260405_144137_001.jpg
```

Each episode contains:

- `schema_version`
- `instruction`, `scene_id`, `operator_id`
- optional task metadata such as `task_family`, `target_type`, `target_description`, `target_instance_id`, `task_tags`, `collector_notes`, `instruction_source`
- `frames[]`
- per-frame `state`, `raw_action`, `control_action`, `image`, and timestamps

## Stage 1 collection

Use the dedicated stage1 binary when collecting only the controlled motion-instruction set.

```bash
cd /home/xiaohui/unitree_go2/go2_vla_collector
./native/build/go2_collector_stage1 \
  --network-interface eno1 \
  --scene-id corridor_a \
  --operator-id op_01
```

Stage1 behavior:

- fixed to `single_action`
- auto-labels from the locked motion key
- rejects semantic-task flags such as `--instruction`, `--task-family`, and `--target-*`
- keeps data separate in `data_stage1/`

Controlled instruction set:

- `go forward`
- `move backward`
- `strafe left`
- `strafe right`
- `stand up`
- `lie down`
- `turn left`
- `turn right`
- `stay still`

## General collection

The general collector keeps both legacy motion-only capture and stage2 semantic collection.

Legacy motion-only example:

```bash
cd /home/xiaohui/unitree_go2/go2_vla_collector
./native/build/go2_collector \
  --network-interface eno1 \
  --scene-id corridor_a \
  --operator-id op_01 \
  --instruction "go forward"
```

Semantic trajectory example:

```bash
cd /home/xiaohui/unitree_go2/go2_vla_collector
./native/build/go2_collector \
  --network-interface eno1 \
  --capture-mode trajectory \
  --scene-id corridor_a \
  --operator-id op_01 \
  --instruction "go to the door" \
  --task-family goal_navigation \
  --target-type door \
  --target-description "glass door at corridor end" \
  --target-instance-id corridor_a_door_01 \
  --task-tags indoor,bright_light
```

Additional semantic examples:

```bash
./native/build/go2_collector \
  --network-interface eno1 \
  --scene-id hall_b \
  --operator-id op_01 \
  --instruction "follow the person in front of you" \
  --task-family visual_following \
  --target-type person \
  --target-description "adult with black jacket"
```

```bash
./native/build/go2_collector \
  --network-interface eno1 \
  --scene-id lab_boxes \
  --operator-id op_01 \
  --instruction "go around the obstacle and continue forward" \
  --task-family obstacle_aware_navigation \
  --target-type obstacle \
  --target-description "cardboard box blocking center path"
```

Collector notes:

- `scene_id` and `operator_id` are always required
- one semantic collection run should keep a stable task configuration
- if `--instruction` is omitted in the general collector, it falls back to the motion label
- `task_tags` is optional CSV metadata for later filtering

Capture modes in the general collector:

- `single_action`
  - press `R` to arm
  - the first single motion key locks the motion label
  - logging starts after 0.5s
  - mixed input or direction changes discard the segment
  - key release auto-ends the segment
- `trajectory`
  - press `R` to start recording immediately
  - multi-stage motion changes are allowed
  - press `T` to end and save
  - press `ESC` to discard
  - `--instruction` is required

## Keyboard controls

- `W/S`: forward or backward
- `A/D`: strafe left or right
- `Q/E`: turn left or right
- `R`: start the current capture flow
- `T`: stop the current capture segment and save
- `ESC`: cancel the current armed or active segment
- `Space`: emergency stop
- `C`: clear fault or toggle stand up/down
- `P`: print status
- `H`: print help
- `X`: quit

## Offline inspection and validation

Generate the HTML replay report:

```bash
cd /home/xiaohui/unitree_go2/go2_vla_collector
python3 scripts/sanity_check_dataset.py --data-root data --output-dir outputs/sanity_check
```

Validate trainability:

```bash
cd /home/xiaohui/unitree_go2/go2_vla_collector
python3 tools/validate_bc_dataset.py --dataset-root data --report-path outputs/validate_report.json
```

The validator reports dataset issues such as:

- missing metadata
- image or timestamp mismatches
- instruction imbalance
- low scene/target overlap for repeated instructions
- lack of visually necessary task families

## Convert and train

Convert raw sessions into manifests:

```bash
cd /home/xiaohui/unitree_go2/go2_vla_collector
python3 tools/convert_llada_vla_dataset.py --raw-root data --output-root data/llada_vla_converted --overwrite
```

Run the lightweight baseline:

```bash
cd /home/xiaohui/unitree_go2/go2_vla_collector
python3 tools/llada_vla_baseline.py \
  --dataset-root data/llada_vla_converted \
  --save-model data/llada_vla_converted/baseline_model.json
```

Run the multimodal diagnostic baseline:

```bash
cd /home/xiaohui/unitree_go2/go2_vla_collector
python3 tools/llada_vla_baseline_v2.py \
  --dataset-root data/llada_vla_converted \
  --output-dir outputs/baseline_v2_goal_nav \
  --ablation-mode image_plus_instruction
```
